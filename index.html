Role Analysis

직무의 본질

1. 구축

신규 FAB의 Track, Stocker, Port, 차량 수, 물동량을 사전에 검증하고 구축 프로젝트를 관리합니다.

2. 제어

MES 요청을 MCS/AMHS 로직으로 변환해 배차, 경로, 우선순위, Port 연계를 실시간으로 결정합니다.

3. 소프트웨어

Dispatching, Routing, Traffic Control, Deadlock 방지, 예외 복구 로직을 개발하고 검증합니다.

4. 운영 최적화

로그와 현장 이벤트를 분석해 Queue, 반송 시간, 장비 Idle, 장애 복구 시간을 줄입니다.

System Context

MES와 물류 장비 사이의 중간 제어 계층

MES는 생산 진행과 다음 공정 정보를 알고, AMHS/MCS는 실제 반송 가능한 자원과 경로 상태를 압니다. IT(AMHS)는 두 세계를 동기화해 생산 지시가 물리적 이동으로 안정적으로 실행되게 만듭니다.

  1. MESLot 상태, 공정 순서, 우선순위 요청
  2. MCS / AMHS배차, 경로, Traffic Rule, 상태 동기화
  3. OHT / AMR / ConveyorFOUP 이동, 센서 이벤트, 장애 보고
  4. Stocker / Tool Port임시 보관, 투입 대기, 장비 Idle 영향
  5. Data Platform반송 시간, Queue, 장애, 개선 효과 분석

강한 Fit

차별화 포인트

6DOF 시뮬레이션과 Python 기반 분석 경험은 신규 FAB 물동량 검증, 병목 예측, 제어 로직 사전 검증에 바로 연결됩니다.

보완 영역

반도체 FAB 용어, MES-MCS-AMHS 인터페이스, OHT/Stocker/Port 제약, 24시간 운영 장애 대응 관점을 학습해야 합니다.

Operational Playbook

AMHS 엔지니어의 문제 해결 절차

01

현상 정의

어느 구간, 어느 시간대, 어느 Lot군에서 Queue와 지연이 발생하는지 분리합니다.

02

상태 동기화 확인

MES, MCS, 장비 이벤트의 Lot 위치와 완료 시점이 일치하는지 검증합니다.

03

병목 원인 모델링

Track 혼잡, Port 부족, Stocker 포화, 차량 부족, Rule 충돌 중 지배 원인을 찾습니다.

04

개선안 시뮬레이션

차량 수, 우회 경로, 우선순위 정책, Buffer Rule을 바꿔 KPI 효과를 비교합니다.

05

점진 적용

운영 리스크가 낮은 구간부터 Rule을 적용하고 반송 시간과 장애 건수를 추적합니다.
